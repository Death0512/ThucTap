01 — Objective

The problem we set out to solve

The platform scraped Facebook with raw Playwright. The goal: adopt the adaptive scraping framework Scrapling for stronger stealth — without breaking the extraction pipeline.

Key constraint discovered

Facebook does not serve content as HTML. It streams data through internal GraphQL API calls. The scrapers work by intercepting network responses (page.on("response")) and walking the JSON — not by parsing the DOM.

Why "pure Scrapling" was rejected

  • Scrapling cannot intercept GraphQL network responses
  • Its strength (adaptive HTML selectors) doesn't apply to JSON
  • A full rewrite would discard the most reliable data path

Decision: a hybrid — each library used only where it is strictly better.

02 — Architecture

FBSession yields a plain Playwright Page, so every existing goto / on / evaluate / screenshot call works unchanged. Scrapling owns acquisition; Playwright owns data capture.

05 — Implementation

Minimal, surgical change

Each scraper swapped only its browser-launch block. The data loop is untouched.

● Before

with sync_playwright() as pw:
  browser, ctx = pw_utils.launch_browser(pw)
  page = ctx.new_page()
  pw_utils.login(page, COOKIE_FILE)
  ... scrape loop ...
  browser.close()

● After

with FBSession(
       cookie_file=COOKIE_FILE,
       headless=True) as page:
  ... same scrape loop ...
  # context closes automatically
06 — Infrastructure

Docker: eliminated a redundant browser download

Root cause

scrapling install internally runs playwright install chromium. The Dockerfile was installing Chromium twice.

Fix

  • Collapsed to a single scrapling install
  • One Chromium serves Scrapling + report render
  • PLAYWRIGHT_BROWSERS_PATH set before install

Outcome: smaller image, faster build, no install/runtime path ambiguity.

08 — Conclusion

What the hybrid achieves

Gains

  • Scrapling-grade stealth & fingerprint spoofing
  • GraphQL JSON extraction fully retained
  • Zero change to extraction / analysis logic
  • Leaner, faster, unambiguous Docker build

Engineering principle

Adopt a new tool for what it does better — not as a wholesale replacement. The right boundary preserved a working system while upgrading its weakest layer.
